Misty Programming Language:Misty Machine Code

Misty Machine

The Misty Machine is an abstract executor of Misty executables. It could be directly implemented in hardware, and perhaps someday it will, but in the meantime, it is used to define an intermediate representation that can be interpreted or fed to backend code generators for real machines.

Misty Assembly Language

The format of Misty machine code is JSON. A JSON object contains fields that describe various aspects of the program, including an instructions field.

The instructions field contains an array containing labels and instructions.

Labels are the targets of branch instructions. Labels are encoded as simple text literals.

Instructions describe a unit of work. Instructions are encoded as records, including an op field with a value text that names a machine operation. Instruction records can also include up to three operand fields, with values that are tokens. The names and meanings if the operands depends on the op. The use of tokens as operands facilitates debugging and error reporting.

{
    name: "your_name_here",
    data: {...}
    instructions: [
        "label",
        {
            op: ...,
            x: ...,
            y: ...,
            z: ...
        },
        ....
    ]
}
